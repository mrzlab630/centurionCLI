### Rules
- `content` is HTML (supports full HTML5: h1-h6, p, table, ul/ol, hr, b/i/em, code, pre, blockquote, img, style)
- `password` — generate a memorable 4-word password (e.g., "sol-sniper-peak-42")
- `delete_after_reading` — 30 for standard reports (30 views), 1 for one-time secrets
- Max content: no hard limit documented, but keep under 500KB for reliability
- Images: use base64 data URIs (`<img src="data:image/png;base64,...">`) for charts

## Report Formatting Standards

### Structure
Every report MUST follow this structure:

```html
<!-- 1. Header -->
<div style="background: linear-gradient(135deg, #1a1a2e 0%, #16213e 100%);
            color: #fff; padding: 24px; border-radius: 12px; margin-bottom: 20px;">
  <h1 style="margin:0; font-size:28px;">🔮 REPORT TITLE</h1>
  <p style="margin:8px 0 0; opacity:0.8; font-size:14px;">
    Date: YYYY-MM-DD | Source: LEGIONARY_NAME | Data: N pools, M trades
  </p>
</div>

<!-- 2. Executive Summary (3-5 bullet points) -->
<div style="background:#f0f9ff; border-left:4px solid #3b82f6; padding:16px;
            border-radius:0 8px 8px 0; margin-bottom:20px;">
  <h2 style="margin:0 0 8px; color:#1e40af;">📋 Key Findings</h2>
  <ul style="margin:0; padding-left:20px;">
    <li>Finding 1</li>
    <li>Finding 2</li>
  </ul>
</div>

<!-- 3. Charts (base64 images) -->
<div style="text-align:center; margin:20px 0;">
  <img src="data:image/png;base64,..." style="max-width:100%; border-radius:8px;
       box-shadow:0 2px 8px rgba(0,0,0,0.1);" />
  <p style="color:#666; font-size:12px; margin-top:4px;">Caption</p>
</div>

<!-- 4. Data Tables -->
<table style="width:100%; border-collapse:collapse; margin:20px 0;">
  <thead>
    <tr style="background:#1e293b; color:#fff;">
      <th style="padding:10px; text-align:left;">Column</th>
    </tr>
  </thead>
  <tbody>
    <tr style="border-bottom:1px solid #e2e8f0;">
      <td style="padding:8px;">Value</td>
    </tr>
  </tbody>

<!-- 5. Footer -->
<hr style="border:none; border-top:2px solid #e2e8f0; margin:30px 0;">
<p style="color:#94a3b8; font-size:12px; text-align:center;">
  ⚔️ Generated by LEGION | TABULARIUS Publisher | DISCIPLINA ET FIDES
```

### Image Embedding
Charts saved as PNG files must be converted to base64 for embedding:

```bash
# Convert PNG to base64 data URI
base64 -w0 /tmp/chart.png | sed 's/^/data:image\/png;base64,/'
```

Then embed: `<img src="data:image/png;base64,AAAA..." />`

**IMPORTANT**: If base64 is too large (>300KB per image), resize first:
```bash
python3 -c "
from PIL import Image
img = Image.open('/tmp/chart.png')
img.thumbnail((1200, 800))
img.save('/tmp/chart_small.png', optimize=True, quality=85)
"
```

## Publishing Workflow

### Step 1: Collect Data
- Read all analysis output (text, tables, numbers)
- Find all chart files (PNG) referenced in the analysis
- Identify the source legionary and analysis type

### Step 2: Format HTML
- Transform raw output into styled HTML using the template above
- Embed charts as base64 images
- Format tables with proper styling
- Add executive summary with key findings
- Add emoji for visual appeal

### Step 3: Publish
```bash
# Generate password
PASSWORD="legion-$(date +%s | tail -c 5)-$(openssl rand -hex 2)"

# Publish
curl -s -X POST "https://tinynotepad.com/api/note/create-secure" \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer Gd4YHiPoQJkpjsg6_7yQHVxXrvQcI_Kk-jh6byVuIZ7JXWXI" \
  -d "{
    \"content\": \"$(cat /tmp/report.html | python3 -c 'import sys,json; print(json.dumps(sys.stdin.read())[1:-1])')\",
    \"password\": \"$PASSWORD\",
    \"delete_after_reading\": 30
  }"
```

### Step 4: Return Summary
After publishing, return to the user:

```
📜 Report published!

**KEY FINDINGS:**
1. [Finding 1]
2. [Finding 2]
3. [Finding 3]

📎 Full report: https://tinynotepad.com/n/SLUG
🔑 Password: `PASSWORD`
```

## Anti-Patterns

### DON'T
- Publish raw markdown (must be HTML)
- Skip the executive summary
- Embed uncompressed 5MB images
- Use plain unstyled tables
- Forget to return the password to user
- Publish without a descriptive title

### DO
- Always compress images before base64 encoding
- Always generate a unique password per report
- Always include date + data volume in header
- Always return summary + link + password
- Use consistent color scheme
- Test that HTML renders correctly
