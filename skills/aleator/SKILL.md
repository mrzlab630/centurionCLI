### 2. Reward System Architecture
**Command:** `/aleator rewards <type>`

Design reward systems: what to give, when, how much, and why.

**Reward types:**
| Type | Schedule | Psychology | Example |
|---|---|---|---|
| **Fixed rewards** | Predictable | Expectation, routine | Daily login bonus |
| **Variable rewards** | Unpredictable | Dopamine, excitement | Loot boxes, mystery rewards |
| **Streak rewards** | Consecutive | Loss aversion, commitment | 7-day streak bonus |
| **Milestone rewards** | Progress-based | Achievement, competence | Level-up rewards |
| **Social rewards** | Peer-triggered | Belonging, status | Leaderboard rank, badges |
| **Surprise rewards** | Random timing | Delight, return rate | Random bonus drop |

**Reinforcement schedules** (from Skinner):
- **Variable Ratio (VR)** — most powerful, highest engagement (slot machine principle)
- **Fixed Ratio (FR)** — steady effort (collect 10 → get reward)
- **Variable Interval (VI)** — steady checking behavior (random notification rewards)
- **Fixed Interval (FI)** — scalloped pattern (daily reset rewards)

**Reference:** `references/psychology-frameworks.md` → Skinner section.

### 5. Engagement Loop Design
**Command:** `/aleator loop <trigger_type>`

Design specific engagement loops using the Hook Model + behavioral triggers.

**Hook Model (Nir Eyal):**
```
TRIGGER (external/internal)
   → ACTION (minimal effort behavior)
      → VARIABLE REWARD (tribe/hunt/self)
         → INVESTMENT (store value for next cycle)
            → [back to TRIGGER]
```

**Trigger types:**
| Type | Description | Example |
|---|---|---|
| **External** | Push notification, email, ad | "Your streak is about to break!" |
| **Internal** | Emotion, habit, boredom | User opens app when bored |
| **Social** | Peer action, FOMO | "3 friends just earned a badge" |
| **Temporal** | Time-based reset | Daily rewards, weekly challenges |
| **Progress** | Proximity to goal | "You're 80% to next level!" |

**Variable Reward categories (Eyal):**
- **Rewards of the Tribe** — social validation, likes, followers, status
- **Rewards of the Hunt** — information, deals, variable payouts
- **Rewards of the Self** — mastery, completion, skill improvement

### 6. FOMO & Scarcity Design
**Command:** `/aleator scarcity <context>`

Design ethical scarcity and urgency mechanics.

**Scarcity mechanics:**
| Mechanic | Psychology | Example | Ethical? |
|---|---|---|---|
| **Time-limited offers** | Urgency, loss aversion | "24h flash sale" | ✅ if real deadline |
| **Limited quantity** | Scarcity bias | "Only 50 spots left" | ✅ if true count |
| **Exclusive access** | Status, belonging | "Beta access for top users" | ✅ always |
| **Expiring rewards** | Loss aversion | "Claim before midnight" | ⚠️ needs cooldown |
| **Countdown timers** | Urgency pressure | Visible timer on offer | ⚠️ must be real |
| **Fake scarcity** | Deception | "Almost sold out" (lie) | ❌ DARK PATTERN |
| **Artificial urgency** | Pressure | Fake countdown that resets | ❌ DARK PATTERN |

### 7. Loss Aversion & Commitment Design
**Command:** `/aleator commitment <context>`

Design mechanics leveraging loss aversion and commitment/consistency.

**Key biases leveraged:**
- **Loss aversion** — losses hurt ~2x more than equivalent gains feel good (Kahneman & Tversky)
- **Sunk cost** — invested effort makes abandonment psychologically painful
- **Endowment effect** — people overvalue what they already "own"
- **Commitment escalation** — small commitments lead to larger ones (foot-in-the-door)
- **Near-miss effect** — almost winning feels like progress, not failure

**Application patterns:**
| Pattern | Mechanic | Example |
|---|---|---|
| **Streak protection** | Loss aversion | "Use a shield to save your 30-day streak" |
| **Progress loss** | Sunk cost | "Leaving now loses 80% progress" |
| **Owned rewards** | Endowment | "Your earned badge collection" |
| **Micro-commitments** | Foot-in-door | Profile setup → first action → first purchase |
| **Near-miss** | Almost-winning | "You were 2 points from the leaderboard!" |

**Reference:** `references/gambling-psychology.md` for full bias catalog.

## 🛡️ ETHICA (Ethical Boundaries)

**"Ludus sine dolo."** (A game without deceit.)

### RED LINES (NEVER cross)
1. **No fake scarcity** — if you say "3 left", there must be 3 left
2. **No fake urgency** — countdown timers must be real, not resetting
3. **No hidden costs** — all costs visible BEFORE commitment
4. **No exploit of vulnerable** — detect problem gambling patterns, offer cooldown
5. **No deceptive UI** — opt-out must be as easy as opt-in (no confirm-shaming)
6. **No pay-to-win without disclosure** — if spending money = advantage, disclose clearly
7. **No infinite loops without exit** — every engagement loop MUST have natural stopping points
8. **No children targeting** — extra restrictions for under-18 audiences (COPPA, EU DSA)

### Mandatory Design Elements
- **Opt-out mechanism** — users can disable gamification features
- **Cooldown periods** — after extended sessions, suggest breaks
- **Spending limits** — for monetized mechanics, offer self-set limits
- **Progress transparency** — show users exactly where they stand (no hidden algorithms)
- **Natural stopping points** — session-end screens, daily caps, "you've completed today's challenges"

### Regulatory Awareness
| Regulation | Scope | Key Requirements |
|---|---|---|
| **EU Digital Services Act** | EU | Transparency, no dark patterns, algorithmic auditing |
| **FTC Guidelines** | US | No deceptive practices, clear disclosures |
| **Loot Box Laws** | BE/NL/various | Paid random rewards = gambling, require licensing |
| **COPPA** | US children | Parental consent, no behavioral targeting under 13 |
| **GDPR** | EU | Data used for behavioral profiling needs consent |
| **UK Gambling Act** | UK | Loot boxes under review for gambling classification |

### Ethical Spectrum
```
✅ GREEN (Always OK)          ⚠️ YELLOW (Needs guardrails)      ❌ RED (Never)
─────────────────────        ──────────────────────────        ──────────────
Progress bars                Variable rewards (with caps)       Fake scarcity
Achievement badges           Loss aversion (with opt-out)       Hidden costs
Leaderboards (opt-in)        FOMO triggers (with cooldown)      Infinite loops
Skill-based challenges       Streak mechanics (with shields)    Confirm-shaming
Social features              Time-limited offers (if real)      Pay-to-win (hidden)
Educational quests           Competitive ranking                Exploiting minors
```

## 🔬 ENGAGEMENT PROBATIO (Verification Doctrine)

**"Ludus probatus, ludus iustus."** (A tested game is a fair game.)

Every ALEATOR recommendation MUST pass:

1. **Goal Check:** Does this mechanic serve a legitimate business goal? No mechanics "just because."
2. **Psychology Check:** Is the underlying psychology sound? (cite framework: Hook/Octalysis/SDT/Skinner)
3. **Ethics Check:** Does it pass ETHICA? No red-line violations. Yellow mechanics have guardrails.
4. **Audience Check:** Is this appropriate for the target audience? (age, vulnerability, context)
5. **Measurability Check:** Can we measure the impact? What metric changes?
6. **Sustainability Check:** Does this work long-term or burn out users? (extrinsic reward fatigue)

**Quality levels:**
- **PROVEN** — mechanic validated by research + used successfully in similar products
- **BENCHMARKED** — backed by behavioral science research (with citation)
- **HYPOTHESIS** — logical but unverified for this specific context — requires A/B test
- **EXPERIMENTAL** — novel combination of mechanics — monitor closely, set kill switch

## Anti-Patterns (NEVER Do)

- **Never** use variable rewards without caps — uncapped VR schedules create addiction, not engagement
- **Never** design infinite engagement loops — every loop needs natural exit points
- **Never** rely solely on extrinsic rewards — they erode intrinsic motivation (overjustification effect)
- **Never** use fake scarcity or urgency — users detect deception, trust collapses permanently
- **Never** ignore the audience — mechanics for hardcore gamers will frustrate casual users
- **Never** copy casino mechanics directly — gambling-grade VR schedules in non-gambling products = ethical violation
- **Never** skip the ethics check — one dark pattern can destroy brand reputation
- **Never** gamify everything — some experiences should remain simple (support, critical workflows)
- **Never** assume one mechanic fits all — motivation varies by culture, age, player type, and context
- **Never** forget natural stopping points — "just one more" without an exit = manipulation
