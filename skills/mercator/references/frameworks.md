# MERCATOR — Marketing Frameworks Reference

## Growth & Acquisition

### AARRR (Pirate Metrics) — Dave McClure
```
Acquisition  → How do users discover us? (channels, CAC, attribution)
Activation   → Do they experience core value quickly? (time-to-value, aha moment)
Retention    → Do they come back? (D1/D7/D30 retention, DAU/MAU)
Referral     → Do they invite others? (K-factor, viral coefficient)
Revenue      → Do they generate value? (LTV, ARPU, conversion rate)
```
**When to use:** Diagnosing WHERE in the funnel the problem is. Always start here.

### Growth Loops — Reforge
```
New User → Uses Product → Creates Output → Attracts New Users → ...
```
Types:
- **Content loop:** User creates content → indexed → discovered → new user
- **Referral loop:** User invites → friend joins → friend invites → ...
- **Paid loop:** Revenue → reinvest in ads → new users → revenue → ...
- **Product loop:** User activity → improves product → attracts users

**When to use:** Designing self-sustaining growth mechanisms.

### Bullseye Framework — Gabriel Weinberg
```
1. Brainstorm ALL 19 traction channels
2. Rank by potential (inner ring = 3 best)
3. Test inner ring cheaply
4. Double down on winner
```
19 channels: viral marketing, PR, SEO, content marketing, email, engineering as marketing,
targeting blogs, business development, sales, affiliate, existing platforms, trade shows,
offline events, speaking, community building, unconventional PR, SEM, social ads, display ads.

**When to use:** Choosing acquisition channels. Don't spray — focus.

## Engagement & Retention

### Hook Model — Nir Eyal
```
External/Internal Trigger → Action (simplest behavior) →
Variable Reward (tribe/hunt/self) → Investment (stored value)
```
- **Trigger:** Push notification, email, internal craving
- **Action:** Open app, check price, make a trade
- **Variable Reward:** P&L surprise, new alpha, community reaction
- **Investment:** Saved settings, watchlists, trade history, reputation

**When to use:** Designing habit-forming features.

### Behavioral Nudge Theory — Thaler & Sunstein
Key nudges for digital products:
- **Default bias:** Opt-in by default (notifications, alerts)
- **Social proof:** "1,247 traders active right now"
- **Loss aversion:** "You missed 3 opportunities while away"
- **Anchoring:** Show potential gains before asking for action
- **Scarcity:** "Limited spots for beta feature"
- **Progress effect:** "You're 60% set up — complete your profile"

**When to use:** Designing onboarding, re-engagement, feature adoption flows.

### Gamification Framework
Elements (pick 2-3, never all):
- **Points/XP** — track activity
- **Levels/Ranks** — progression (Bronze → Silver → Gold → Diamond)
- **Achievements/Badges** — milestones (First Trade, 100 Trades, etc.)
- **Leaderboards** — competition (top traders this week)
- **Streaks** — consistency (7-day trading streak)
- **Unlocks** — gated features (reach level X to unlock Y)

**When to use:** Increasing retention and engagement depth.

## Messaging & Copywriting

### PAS (Problem-Agitate-Solve)
```
Problem:  "Tired of missing trades because you're not watching 24/7?"
Agitate:  "Every minute you're away, top wallets are making moves you'll never see."
Solve:    "Copy-trade automatically. Set it once, profit while you sleep."
```

### BAB (Before-After-Bridge)
```
Before: "Manually tracking 50 wallets, missing 90% of trades."
After:  "Every trade copied instantly, zero effort, 24/7."
Bridge: "Our copy-trading bot watches for you — set your wallets, done."
```

### AIDA
```
Attention: Bold claim or shocking stat
Interest:  How/why it's relevant to THEM
Desire:    Paint the outcome they want
Action:    Clear, single CTA
```

### 4U Formula
Every headline/hook should be:
- **Useful** — what's in it for the reader?
- **Urgent** — why act now?
- **Unique** — what's different about this?
- **Ultra-specific** — concrete details, numbers, timeframes

### "So What?" Test
For every feature: ask "So what does that mean for the user?"
- Feature: "We support Raydium, Meteora, PumpFun, Orca"
- So what: "Trade on ANY Solana DEX from one bot — no switching tools"

## Competitive Analysis

### Porter's Five Forces (Adapted for Crypto Bots)
1. **Competitive Rivalry** — how many similar bots exist?
2. **Threat of New Entrants** — how easy to launch a new bot?
3. **Buyer Power** — how easy for users to switch?
4. **Supplier Power** — RPC providers, DEX access, blockchain dependency
5. **Substitute Threat** — DEX UIs, wallet apps, manual trading

### Positioning Canvas
```
For [target user]
Who [has this problem/need]
Our [product] is a [category]
That [key benefit]
Unlike [competitor/alternative]
We [key differentiator]
```

## Campaign Design

### Drip Campaign Principles
- **Timing:** Increase intervals (Day 0, 2, 5, 10, 21, 45)
- **Value-first:** Each message must give value, not just ask for action
- **Escalation:** Start soft (info) → medium (social proof) → strong (incentive/FOMO)
- **Exit criteria:** Remove from sequence when user takes desired action
- **Segment-specific:** Different sequences for different user segments

### A/B Testing Framework
```
1. Hypothesis: "Changing X will improve Y by Z%"
2. Variable:   ONE thing at a time (copy, timing, CTA, audience)
3. Sample:     Calculate minimum N for statistical power (usually ≥100 per variant)
4. Duration:   Minimum 7 days (account for day-of-week effects)
5. Measure:    Primary metric + guardrail metrics
6. Decide:     p < 0.05 = significant. Otherwise = inconclusive, NOT "no effect"
7. Document:   Record in campaign-history.md regardless of outcome
```
